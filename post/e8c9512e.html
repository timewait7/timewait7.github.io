Git常用命令

Posted on 2024-11-28 In Git Word count in article: 1.6k Reading time ≈ 6 mins.

Git 常用命令：Reference

git help

命令 说明
git <verb> --help 查看帮助文档
git <verb> -h 查看快速参考

git config

根据作用范围，git有如下三种配置文件：

配置文件 命令 说明
/etc/gitconfig git config --system 包含系统上所有用户的配置
~/.gitconfig git config --global 只包含当前用户的配置
.git/config git config --local 只包含当前仓库的配置

和配置有关的常用命令：

命令 说明
git config user.name 查看单个配置
git config --list 列出所有配置
git config --list --show-origin 列出所有配置及配置所在的文件
git config --global use.name "zhang3" 配置用户名
git config --global use.email "xxx@xxx.com" 配置邮箱
git config --global alias.co checkout 配置别名

git add

命令 说明
git add <file>... 跟踪/暂存文件

git restore

命令 说明
git restore --staged <file>... 取消暂存
git restore <file>... 撤销修改

git status

git status的作用是查看文件的状态，但该命令的输出较为繁琐，可以加上-s选项简化输出。

git status -s的输出中有两栏表示状态，左侧是暂存区状态，右侧是工作区状态：

  • ??：新文件，未跟踪
  • A ：新文件，已跟踪
  • M：旧文件，已修改未暂存
  • M ：旧文件，已修改已暂存
  • MM：旧文件，修改并暂存后再次修改

git diff

命令 说明
git diff <file>... 查看已修改未暂存的改动
git diff --staged <file>... 查看已暂存未提交的改动

git log

命令 说明
git log 查看提交历史
git log -n 查看最近n次提交历史
git log -p 查看每次提交引入的差异
git log --stat 查看提交的简略统计信息
git log --graph 图形显示分支合并历史
git log --since="2020-10-10" --until="2024-10-10" 显示指定时间范围内的提交
git log --grep='commit' 搜索提交说明中包含指定关键字的提交
git log --name-only 查看已修改的文件清单
git log --name-status
git log --author='zhang3' 查看指定用户的提交
git log --pretty=formt:"%h - %an, %cd: %s" --date=short 自定义输出格式
git log --oneline --decorate 查看当前各分支所指的提交对象
git log --oneline --decorate --graph 查看提交历史、各个分支的指向以及分支分叉情况
git log master..dev 查看在dev分支但不在master分支的提交
git log ^master dev test 查看在dev和test分支但不在master分支的提交
git log dev test --not master 查看在dev和test分支但不在master分支的提交
git log origin/master..HEAD 查看即将推送到远端的提交
git log master...dev 查看只在其中一个分支上的提交

统计代码行数：

git reflog

查看HEAD或分支引用指向的历史

git show

命令 说明
git show HEAD^ 查看HEAD的第1个父提交
git show HEAD~ 查看HEAD的第1个父提交
git show HEAD^2 查看HEAD的第2个父提交
git show HEAD~2 查看HEAD的第1个父提交的父提交
git show HEAD^2~ 查看HEAD的第2个父提交的父提交

git remote

命令 说明
git remote -v 查看远程仓库及其url
git remote add <remote-name> <url> 关联远程仓库
git remote rename <remote-name> <another-name> 重命名远程仓库名
git remote rm <remote-name> 删除关联

git tag

命令 说明
git tag 列出已有标签
git log -l 'v1.0*' 列出特定标签
git log -a v1.0 打附注标签
git log v1.0-lw 打轻量标签
git log -a v0.9 <commit-id> 给历史提交打标签
git push origin v1.0 推送标签

git branch

命令 说明
git branch dev 创建分支
git checkout dev 切换分支
git checkout -b dev 创建并切换分支
git checkout -b dev origin/dev 从远程分支创建分支
git checkout --track origin/dev 从远程分支创建分支
git branch -u origin/dev 设置本地分支跟踪远程分支
git branch -v 查看每一个分支的最后一次提交
git branch -vv 查看设置的所有跟踪分支
git branch --merged 查看已合并分支
git branch --no-merged 查看未合并分支
git branch -d dev 删除分支
git branch -D dev 强制删除分支
git push origin --delete dev 删除远程分支

git merge

fast-forward：当前所在分支指向的提交是被合并分支所指向的提交的直接上游，合并操作只简单的将让当前分支指向被合并分支所指向的提交，这种合并就叫做 fast-forward

git rebase

git co dev
git rebase master
git co master
git merge dev

或者

git rebase master dev（如果当前分支不是dev，此命令会切到dev分支）
git co master
git merge dev

git rebase –onto master server client：选中在client分支但不在server分支的修改，将它们在master分支重放

最佳实践：只把变基命令用于推送前整理提交历史，并且只在从未推送至公共仓的提交上执行变基命令。

git stash

命令 说明
git stash 储藏修改
git stash save '描述' 储藏修改并命名
git stash list 查看储藏列表
git stash apply 应用储藏
git stash apply --index 如果储藏包含已暂存的文件，需加–index参数
git stash drop 删除储藏
git stash pop 应用并删除储藏
git stash --keep-index 只储藏未暂存的文件
git stash -u stash默认只储藏已跟踪的文件，想要储藏未跟踪的文件，需加上-u参数
git stash branch <branch_name> 从储藏创建分支

git reset

命令 说明
git reset --soft HEAD~ 回退到上一个提交，暂存区和工作目录不变
git reset [--mixed] HEAD~ 默认模式，回退到上一个提交，取消暂存，保留工作目录的修改
git reset --hard HEAD~ 回退到上一个提交，撤销工作目录的所有修改

git reset 会同时移动 HEAD 和分支的指向，git checkout 只移动 HEAD
