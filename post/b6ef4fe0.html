Git常用命令

Posted on 2024-08-27 Edited on 2024-11-29 In Git Word count in article: 816 Reading time ≈ 3 mins.

Git基础

文件状态变换

通用命令

命令 说明
git <verb> --help 查看帮助文档
git <verb> -h 查看快速参考

git config

根据作用范围，git有如下三种配置文件：

配置文件 设置命令 说明
/etc/gitconfig git config --system 包含系统上所有用户的配置
~/.gitconfig git config --global 只包含当前用户的配置
.git/config git config --local 只包含当前仓库的配置

和配置有关的常用命令：

命令 说明
git config --list 列出所有配置
git config --list --show-origin 列出所有配置及配置所在的文件
git config user.name 查看某一项配置
git config --global use.name "zhang3" 配置用户名
git config --global use.email "xxx@xxx.com" 配置邮箱
git config --global alias.co checkout 配置别名

git status

git status的作用是查看文件的状态，但该命令的输出较为繁琐，可以加上-s选项简化输出。

git status -s的输出中有两栏表示状态，左侧是暂存区状态，右侧是工作区状态：

  • ??：新文件，未跟踪
  • A ：新文件，已跟踪
  • M：旧文件，已修改未暂存
  • M ：旧文件，已修改已暂存
  • MM：旧文件，修改并暂存后再次修改

文件修改

命令 说明
git add <file> 跟踪/暂存文件
git reset HEAD <file> 取消暂存
git checkout -- <file> 撤销修改

git diff

如果想知道具体修改了哪些文件，可以使用git diff命令：

  • git diff：查看已修改未暂存的内容
  • git diff --staged：查看已暂存未提交的内容
  • git diff --shortstat "@{n day ago}"：查看修改、新增、删除代码行数

git log

命令 说明
git log 查看提交历史
git log -n 查看最近n次体检历史
git log -p 查看每次提交引入的差异
git log --name-only 查看已修改的文件清单
git log --name-status
git log --author='zhang3' 查看指定用户的提交
git log --pretty=formt:"%h - %an, %cd: %s" --date=short 自定义输出格式

远程仓库

命令 说明
git remote -v 查看远程仓库及其url
git remote add <remote-name> <url> 关联远程仓库
git fetch <remote-name> 从远程仓库拉取
git push <remote-name> <branch-name> 推送到远程仓库

git tag

命令 说明
git tag 列出已有标签
git log -l 'v1.0*' 列出特定标签
git log -a v1.0 打附注标签
git log v1.0-lw 打轻量标签
git log -a v0.9 <commit-id> 给历史提交打标签
git push origin v1.0 推送标签
